# NAME

**dp** -- design patterns

# DESCRIPTION

Design Patterns is a collection of design patterns, collected from various
locations on internet, books, or through personal experience. It does not aim
to be a complete or exhaustive list of algorithms. Instead, it is an
in-progress, never ending project, always growing with time as we uncover new
patterns.

The format of the Design Patterns is to demonstrate conceptual idea with
minimal code for readability. It does not aim to provide implementations in all
possible languages. We believe, that the reader is capable to translate the
algorithm into the language of their preference. As such, most of the code is
written in easy to follow language Go.
